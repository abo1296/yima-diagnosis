# 逸马诊断系统 · 行业定制问题方案

> 每个行业 72 题，其中 40 题跨行业通用，32 题按行业定制。
> 定制题集中在三个高行业敏感性维度：运营标准化、供应链、数字化。

---

## 一、问题分类矩阵

| 维度 | 8题 | 敏感度 | 处理方式 |
|------|:--:|:--:|------|
| 战略 | 8 | 低 | **通用**（战略方法论跨行业） |
| 商业模式 | 8 | 中 | **2题定制**（单店模型、回本周期用行业基准） |
| 运营标准化 | 8 | **高** | **8题全部定制**（每个行业 SOP 完全不同） |
| 组织人才 | 8 | 低 | **通用**（人才问题是共通的） |
| 供应链 | 8 | **高** | **8题全部定制**（供应链结构差异极大） |
| 培训体系 | 8 | 中 | **3题定制**（上岗时长、教材、加盟商培训行业化） |
| 督导体系 | 8 | 中 | **3题定制**（检查重点、神秘顾客、品质标准行业化） |
| 数字化 | 8 | **高** | **8题全部定制**（关键系统依行业而定） |
| 企业文化 | 8 | 低 | **通用**（文化问题是共通的） |

**总计：40 题通用 + 32 题行业定制**

## 二、餐饮行业定制题（32题）

### 运营标准化（8题）

**第 1 题：SOP 覆盖度**
- □ 0分 — 后厨靠师傅手感，前厅靠店长经验，没有书面标准
- □ 1分 — 有关键岗位SOP（如锅底炒制、招牌菜），但覆盖不全、更新不及时
- □ 2分 — 全岗位SOP覆盖（后厨每道菜量化到克/秒/温度 + 前厅服务七步法），定期迭代 + 督导检查执行 + 视频化

**第 2 题：出品一致性**
- □ 0分 — 同一道菜不同门店口味差异大，全看当班师傅
- □ 1分 — 招牌菜能做到大致一致，但边缘菜品和摆盘有差距
- □ 2分 — 全品类出品一致，神秘顾客评分偏差<10%，消费者感知"在哪吃都一样"

**第 3 题：新店开业标准化**
- □ 0分 — 每次开新店都是一场"战役"，没有复用流程
- □ 1分 — 有开店 checklist，但厨房动线/设备布局每家都要重新磨合
- □ 2分 — 标准化开店手册（选址→装修→设备→试菜→开业）+ 模块化厨房 + 45天内交付，新店首月即达标

**第 4 题：日常运营管理**
- □ 0分 — 排班/订货/盘点/卫生全凭店长自己安排
- □ 1分 — 有基本制度，但高峰期出餐速度、翻台率、食材损耗各店差异大
- □ 2分 — 全流程数字化（智能排班、自动订货、损耗率自动预警），店长精力从"救火"转向"经营"

**第 5 题：菜品结构管理**
- □ 0分 — 老板/总厨拍脑袋更新菜单，没有数据依据
- □ 1分 — 有新品研发流程，但老品淘汰缺乏销量/毛利数据分析
- □ 2分 — 菜单工程化（引流款/利润款/形象款分层），季度数据复盘，ABC分类汰换

**第 6 题：门店空间标准化**
- □ 0分 — 每家店装修风格不一，识别度低
- □ 1分 — 有VI手册，但餐桌布局、灯光色温、音乐音量各店执行差异大
- □ 2分 — 完整SI体系（按店型分级：旗舰/标准/外卖店），模块化施工，消费者空间体验数据反馈优化

**第 7 题：店长离职抗风险**
- □ 0分 — 厨师长/店长一走，厨房和前厅都乱，至少1个月才能稳住
- □ 1分 — 有副手制度，但关键菜品仍依赖个人，过渡期2-3周
- □ 2分 — 中央厨房+标准化SOP+厨师长储备池，系统大于个人，换人不影响出品

**第 8 题：运营标准持续优化**
- □ 0分 — 标准定下来就不动了，出了大投诉才改
- □ 1分 — 定期收集门店反馈，年度修订一次SOP
- □ 2分 — 数据驱动（出品耗时/客诉分类/损耗率异常）+ 标杆门店经验萃取 + 季度迭代推送到所有门店

### 供应链（8题）

**第 1 题：采购模式**
- □ 0分 — 各店自行采购，供应商五花八门
- □ 1分 — 核心食材（底料/肉类/油脂）集中采购，蔬菜/调料门店自采
- □ 2分 — 全品类集采 + 战略供应商深度绑定 + 反向定制（按需定规格），成本领先同行

**第 2 题：食材质量保障**
- □ 0分 — 靠供应商自觉，来货看一看就入库
- □ 1分 — 有验收标准（新鲜度/规格/产地），抽检为主
- □ 2分 — 全链路品控（产地→冷链→中央厨房→门店），批次可追溯，不合格自动拒收并触发供应商评级

**第 3 题：仓配体系**
- □ 0分 — 没有中央厨房/仓库，供应商直送各门店
- □ 1分 — 有中央厨房或中心仓，覆盖核心区域，但冷链断链偶发
- □ 2分 — 多温层中央厨房 + 区域分仓 + 冷链配送，覆盖半径200km，日配/隔日配可控

**第 4 题：食材损耗与断货**
- □ 0分 — 经常断货（高峰期缺招牌菜），也经常报损（食材过期/变质）
- □ 1分 — 有订货标准但凭经验，食材成本率波动大（±5%），报损率3-8%
- □ 2分 — AI销量预测 + 智能补货，食材成本率稳定在目标±2%以内，报损率<2%

**第 5 题：供应链成本优势**
- □ 0分 — 没算过，或食材成本率明显高于同行
- □ 1分 — 食材成本率跟同行差不多（火锅35-40%/快餐30-35%）
- □ 2分 — 集采规模+中央厨房让食材成本率低于同行3-5个百分点，且差距在拉大

**第 6 题：核心原料"卡脖子"风险**
- □ 0分 — 核心底料/蘸料依赖单一供应商，没有备选
- □ 1分 — 有两家以上供应商可以切换，但配方外泄风险存在
- □ 2分 — 核心底料自建工厂/中央厨房生产 + 战略储备 + 配方专利保护 + 上游参股

**第 7 题：新城市扩展供应链能力**
- □ 0分 — 供应链是跨区扩张的最大障碍，不敢出去
- □ 1分 — 能覆盖但需要建新仓/找新供应商，周期3-6个月，成本高30%+
- □ 2分 — 中央厨房产能前置规划 + 区域分仓可快速扩展，新城市2个月内供应链就位

**第 8 题：供应链数字化**
- □ 0分 — 全靠电话/微信/Excel，订货靠手写单
- □ 1分 — 有基础进销存，但中央厨房和各门店数据没打通
- □ 2分 — 供应链全链路数字化（采购→中央厨房→配送→门店验收），实时库存可见，异常自动预警

### 数字化（8题）

**第 1 题：系统覆盖**
- □ 0分 — 基本没有数字化，全靠手工
- □ 1分 — 有POS收银 + 基础会员系统
- □ 2分 — 核心系统覆盖（POS + 会员CRM + 进销存 + 巡店系统 + 外卖平台对接）

**第 2 题：数据打通**
- □ 0分 — POS、外卖、会员、进销存各是各的，数据对不上
- □ 1分 — 部分系统有对接，但数据偶尔不一致
- □ 2分 — 一个ID贯穿（点单→会员→进销存→外卖→评价），总部看板实时汇总各店数据

**第 3 题：数据驱动决策**
- □ 0分 — 凭经验决策，没有数据可看
- □ 1分 — 月底看报表（营业额/食材成本率/人效），日常经营靠经验
- □ 2分 — 实时数据看板 + 异常自动预警（食材成本率超标/翻台率骤降/差评激增），从一线到高管看数据决策

**第 4 题：消费者端数字化**
- □ 0分 — 纯线下，没有线上触点
- □ 1分 — 有公众号/小程序（点单/排队/会员积分），但没有运营
- □ 2分 — 完整私域体系（小程序点单 + 会员分层 + 精准发券 + 社群运营 + 评价管理），会员复购率可追踪

**第 5 题：数字化投入**
- □ 0分 — 基本没投入
- □ 1分 — 用过一些SaaS工具，很零散，没有统一规划
- □ 2分 — 有数字化预算 + 专职IT/数字化团队（或稳定的外包伙伴），数字化是战略级投入

**第 6 题：外卖/线上占比**
- □ 0分 — 没做外卖或开了但没人管
- □ 1分 — 外卖占比10-25%，但没有精细化运营（满减/竞价/评价回复靠人工）
- □ 2分 — 外卖占比25%+，双平台精细化运营 + 私域外卖占比在提升，线上线下互相导流

**第 7 题：实时数据可见度**
- □ 0分 — 月底才能看到汇总报表
- □ 1分 — 第二天能看到昨天的营业数据
- □ 2分 — 实时看板（营业额/翻台率/人效/食材成本率/差评即时推送），可下钻到单品/时段/员工

**第 8 题：行业数字化水平自评**
- □ 0分 — 低于行业平均水平
- □ 1分 — 跟大部分同行差不多
- □ 2分 — 数字化是竞争加分项（如智能排队、自助点单、厨房KDS、数据中台）

### 商业模式（2题定制）

**第 1 题：单店盈利模型**（替代通用版 model_1）
- □ 0分 — 没有精细化算过，大概感觉是赚钱的
- □ 1分 — 知道大概毛利率和盈亏平衡点，但单店模型不够标准化（不同店型/区域差异大）
- □ 2分 — 单店模型标准化（坪效/翻台率/人效/食材成本率/回本周期KPI清晰），新店开业前就能模拟预测

**第 6 题：投资回报**（替代通用版 model_6）
- □ 0分 — 没算过回本周期，或远超预期
- □ 1分 — 回本周期在12-18个月（行业常见范围），但不稳定
- □ 2分 — 回本周期稳定在8-12个月以内（优于行业平均），投资回报可预测，投资者排队

### 培训体系（3题定制）

**第 1 题：新店员上岗**（替代通用版 training_1）
- □ 0分 — 3-4周或更长，跟着师傅看，没有系统培训
- □ 1分 — 2-3周，有集中培训但实操仍靠师傅带
- □ 2分 — 1-2周，标准化培训（视频SOP + 模拟操作 + 师傅带教 + 考核认证上岗）

**第 3 题：培训教材**（替代通用版 training_3）
- □ 0分 — 口头传授，没有书面教材
- □ 1分 — 有纸质SOP手册，但没有视频化，更新不及时
- □ 2分 — 全岗位视频SOP库 + 考核题库 + 定期更新，新菜品/新流程上线即更新培训内容

**第 6 题：加盟商培训**（替代通用版 training_6）
- □ 0分 — 收了加盟费就完事，培训走过场
- □ 1分 — 开业前集中培训 + 开业带店7-15天
- □ 2分 — 开业前系统培训（理论+实操+考核）+ 开业后持续赋能（月度复训、新品培训、经营分析辅导），加盟商培训标准与直营一致

---

### 督导体系（3题定制）

**第 2 题：巡店检查重点**（替代通用版 supervision_2）
- □ 0分 — 不定期走马观花，看心情检查
- □ 1分 — 约一个月一次，主要看卫生和出餐速度
- □ 2分 — 有明确频次+分级检查标准（食品安全40%+出品一致性30%+服务体验20%+环境10%），每次出评分报告

**第 7 题：神秘顾客**（替代通用版 supervision_7）
- □ 0分 — 没有
- □ 1分 — 偶尔做，一年1-2次，结果没有系统利用
- □ 2分 — 季度神秘顾客计划 + 结果与内部督导互相验证 + 问题门店专项复检

**第 8 题：品质信心**（替代通用版 supervision_8）
- □ 0分 — 完全没信心，不同门店体验天差地别
- □ 1分 — 大部分店还行（70%达标），小部分店心里没底
- □ 2分 — 有数据支撑的信心——各门店出品评分/客诉率/回头率在可控范围内，90%+门店达标

## 三、零售行业关键差异

零售与餐饮最大的不同：**SKU多、库存周转是核心、鲜食是利润引擎。**

| 维度 | 零售 vs 餐饮的核心差异 |
|------|---------------------|
| 运营标准化 | 陈列（棚割图）、收货/报损/盘点流程、促销执行 替代 出餐标准化 |
| 供应链 | 多温层+日配、鲜食自有品牌、SKU宽度3000+ 替代 中央厨房+冷链 |
| 数字化 | 选址决策引擎、自动补货/智能分货、加盟商合规监控 替代 外卖/排队/厨房KDS |

**仅需修改的关键题**（约15道题的选项文本）：

- operation_1: "SOP" → 商品结构标准化 + 运营流程（开业/收货/陈列/报损/交接班检查表）
- operation_2: "出品一致" → 核心品项铺货率 + 鲜食品质一致性 + 服务话术标准化
- supply_chain_1: "食材采购" → 商品采购（标品集采 + 鲜食本地化 + 自有品牌开发）
- supply_chain_4: "食材损耗" → 鲜食报损率 + 滞销品占比 + 缺货率
- digital_1: "POS+会员" → POS + 自动补货 + 智能分货 + 选址引擎
- digital_6: "外卖占比" → 线上到家占比（京东到家/美团/饿了么/自营小程序）

## 四、医药行业关键差异

医药与餐饮最大的不同：**GSP合规是生存线、执业药师是瓶颈、处方外流是机会。**

| 维度 | 医药 vs 餐饮的核心差异 |
|------|---------------------|
| 运营标准化 | GSP合规（温湿度/处方审核/药师在岗）替代 出餐标准化 |
| 供应链 | 强监管全程可追溯 + 批零一体化 + 自有品牌/贴牌品种 替代 中央厨房+冷链 |
| 数字化 | 处方流转+医保对接、慢病管理CRM、GSP合规自动化 替代 外卖/排队 |

**关键差异题**：

- operation_1: "SOP" → GSP合规细则逐条映射到门店操作 + 处方审核流程 + 药品分类管理
- supply_chain_2: "质量保证" → GSP全程追溯（从批发到门店到消费者），效期预警，温湿度自动监控
- training_1: "上岗" → 执业药师在岗率 + 店员药学知识培训 + GSP法规培训
- digital_1: "系统" → GSP合规系统 + 处方流转+医保对接 + 慢病管理CRM

## 五、教育行业关键差异

教育与餐饮最大的不同：**核心"产品"是人（教师）、预付费资金监管、续费率决定一切。**

| 维度 | 教育 vs 餐饮的核心差异 |
|------|---------------------|
| 运营标准化 | 课程标准化（而非产品）+ 教学交付SOP + 测评标准化 |
| 供应链 | 师资供给（而非食材）+ 场地坪效 + 课程内容开发 |
| 数字化 | 教师画像+排课、学员全生命周期管理、家校互通 替代 外卖/排队 |

**关键差异题**：

- operation_2: "出品一致" → 同一门课不同老师交付体验差异 + 阶段测评结果一致性
- supply_chain_1: "采购" → 教师招聘渠道 + 师资储备池 + 兼职教师管理
- supply_chain_4: "断货" → 教师排课满负荷率 + 名师流失导致的排课缺口
- digital_4: "消费者数字化" → 学员端（在线作业/阶段测评/学习报告）+ 家长端（成长档案/家校沟通）

## 六、服饰行业关键差异

服饰与餐饮最大的不同：**库存是生死线、买手依赖、售罄率决定利润。**

| 维度 | 服饰 vs 餐饮的核心差异 |
|------|---------------------|
| 运营标准化 | 陈列标准化 + 服务七步法（迎宾-探需-搭配-试穿-成交-附加-加微信）替代 出餐标准化 |
| 供应链 | 期货模式+柔供快反、智能配补调、过季库存贬值 替代 中央厨房+冷链 |
| 数字化 | 智能配补调系统、单店商品画像+订货建议、导购赋能工具 替代 外卖/排队 |

**关键差异题**：

- operation_2: "出品一致" → 陈列标准化执行率 + 导购服务七步法执行率
- supply_chain_4: "断货" → 畅销款缺货率 + 滞销款占比 + 季末售罄率
- digital_1: "系统" → 智能配补调系统 + 单店商品画像 + 导购助手

## 七、实施建议

### 技术实现方式

**方案A：问卷引擎支持动态题目（推荐）**

在 questions.ts 中，将每道题标记 `generic: true/false`，行业题目存储在 `industryVariants` 中：

```typescript
interface Question {
  id: string;
  dimension: string;
  generic: boolean; // true = 所有行业通用
  text: string;     // 通用题文字 or 默认行业文字
  industryVariants?: Record<string, {  // 行业定制版本
    text: string;
    options: { value: number; label: string }[];
  }>;
}
```

用户在首页选择行业后，问卷引擎自动加载对应行业版本。

### 分阶段实施

| 阶段 | 内容 | 工作量 |
|------|------|:--:|
| **Phase 1** | 先做餐饮行业定制版（最大用户群），其他行业用通用版 | 1天 |
| **Phase 2** | 零售 + 医药 + 教育 + 服饰 四个行业定制 | 2天 |
| **Phase 3** | 酒类 + 家电定制 | 0.5天 |
| **Phase 4** | 根据数据反馈，持续优化选项描述 | 持续 |

### 核心价值

行业定制后，诊断准确度提升 30-50%，用户感知从"一本通用的体检报告"变成"专门为我们这个行业做的诊断"。这是区分"业余工具"和"专业诊断系统"的关键分水岭。
